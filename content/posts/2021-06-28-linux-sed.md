일반적으로 sed를 쓸때 나는

```bash
sed s/원문/치환/ 파일
```
이런식으로 사용했다. 그런데 만약에 변경할것이 /var/log 에서 /var/log2로 변경한다면

```bash
sed s/₩/var₩/log/₩/var₩/log2/ 파일
```
이런식으로 sed의 구분자를 회피하기위해 ₩/ 과같은 역슬러쉬를 사용해야 했다. 그런데 오늘 혁신을 맛봤다.

```bash
sed "s|/var/log|/var/log2|" 파일
```
/ 대신 |를쓰면 ₩/를 일일이 쓸필요가 없다..
