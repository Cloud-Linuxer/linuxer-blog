# Deployment Instructions

## Step 1: Create GitHub Repository

1. Go to https://github.com/new
2. Repository name: `linuxer-hugo` (or your preferred name)
3. Description: "Linuxer blog - WordPress to Hugo migration"
4. Visibility: Public (required for free GitHub Pages)
5. Do NOT initialize with README (we already have one)
6. Click "Create repository"

## Step 2: Push Code to GitHub

```bash
cd /home/linuxer-hugo

# Rename branch to main (GitHub default)
git branch -M main

# Add GitHub remote (replace USERNAME with your GitHub username)
git remote add origin https://github.com/USERNAME/linuxer-hugo.git

# Push code
git push -u origin main
```

## Step 3: Enable GitHub Pages

1. Go to repository settings: `https://github.com/USERNAME/linuxer-hugo/settings/pages`
2. Under "Source", select: "GitHub Actions"
3. The workflow will automatically trigger on the next push

## Troubleshooting

### Build Fails
- Check GitHub Actions logs for errors
- Ensure Hugo version matches (v0.146.0)
- Verify PaperMod submodule is properly initialized

### Custom Domain Not Working
- Wait 24-48 hours for DNS propagation
- Verify DNS records are correct using `dig linuxer.name`
- Check GitHub Pages settings for DNS verification status

### Site Shows 404
- Ensure baseURL in hugo.toml matches your domain
- Check that GitHub Pages source is set to "GitHub Actions"
- Verify workflow completed successfully
